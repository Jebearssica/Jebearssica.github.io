双向链表, 一个基础数据结构, 上至找工作笔试, 下至大学期末考试都会考到的东西. 概念上称得上简单, 实现上称得上细致.

## 概念

概念上简单是真的简单, 相信接触过一些数据结构的人都能够写出如下代码:

```c++
template <class T>
class list
{
private:
    struct node
    {
        node *prev;
        node *next;
        T data;
    };
    node *root;

public:
    list(/* args */) {}
    ~list() {}
};
```

我们会发现, 其中除开对一个链表的头结点的存储之外, list并不包含其他更多的内容, 而后续的所有操作都是在上述结构下的延展.

> 实际上 `std::list` 中还存了一个链表的迭代器, 用于实现诸多容器遍历.

## 自增运算符重载

首先, 你需要一定的预备知识, 否则你可能一头雾水. 针对于整数的自增运算符有如下特性:

```c++
int i = 1;
// valid: same as ++(++i);
++++i;
// invalid: lvalue required as increment operand
i++++;
```

> 即, `(A++)` 时, 左值 `A` 必须是一个可增加的变量, 不能是常量. 因此, 针对于链表中的迭代器, 我们需要使得其重载的自增运算符与整数自增运算符有相同性质.
{: .prompt-tip }

```c++
// ++A 调用, 对 A 自身进行改变后, 返回该变量的地址
_List_iterator &operator++()
{ // preincrement
    ++(*(_Mybase *)this);
    return (*this);
}

// A++ 调用, 为确保满足上述特性, 传递回的是一个临时变量(未改变前)的拷贝, 不具备可增加的性质
_List_iterator operator++(int)
{ // postincrement
    // 此处先调用的拷贝构造函数, 将 *this 转变成对应形参的类型; 而非先调用 operator*() 直接传递 data
    _List_iterator _Tmp = *this;
    // 同理, 此时先调用 operator++(), 将 *this 转变对应形参类型
    ++*this;
    return (_Tmp);
}
```
